# Ontology (Governance)

## 핵심 메시지
Ontology는 거창한 기술이 아니라,
- 조직의 암묵지를 형식지로 **제대로 드러내고**
- AI Agent 확산 시 불가피한 **품질·일관성 문제**를 조직 차원에서 관리하는
‘업무 기준(용어/규칙/Workflow)’이다.

## 왜 필요한가?
- AI Agent가 늘어나면 품질/일관성 저하는 구조적으로 발생 → 의사결정 오류/재작업/운영 리스크.
- 조직 암묵지는 사람마다 편차가 크고, 이것이 Agent 개발 품질의 병목.

## Bottom-up (권장)
- 완성된 Ontology 모델을 먼저 정의하지 않는다.
- 현업 산출물(JD/Workflow/AX 과제 결과물)에서 공통 용어/규칙/구조를 추출 → 후보화 → 리뷰/승인 → Ontology로 승격.

## 구성 요소(참고)
- Ontology = Data + Logic + Action
